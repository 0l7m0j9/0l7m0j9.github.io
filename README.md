# Project: Playlist Maker

## 프로젝트 개요

사용자가 원하는 무드, 상황, 장르 등을 입력하면 그 조건에 맞는 플레이리스트를 생성해 주는 서비스입니다.

기존 플레이리스트 생성 서비스는 새로운 곡을 추천하는 데 초점이 맞춰져 있지만, 이 프로젝트는 사용자가 이미 알고 있는 곡이나 자신의 플레이리스트에 저장된 곡을 바탕으로 상황에 맞는 음악을 골라주는 것을 목표로 합니다.

## 개발 방향

이 프로젝트는 별도의 백엔드 서버를 두지 않고 GitHub를 중심으로 개발합니다.

초기 버전은 GitHub 저장소와 GitHub Pages를 이용한 정적 웹앱 형태로 구현합니다. 사용자의 플레이리스트 데이터는 서버에 저장하지 않고, 브라우저의 로컬 저장소 또는 사용자가 직접 업로드한 파일을 활용하는 방식으로 시작합니다.

이 방식은 서버 구축과 배포 부담이 적고, 프로젝트를 빠르게 완성해 볼 수 있다는 장점이 있습니다.

## 기존 서비스의 아쉬운 점

TuneMyMusic 같은 서비스는 주제나 키워드를 입력하면 플레이리스트를 생성하거나 유사곡을 찾아주고, 플랫폼 간 곡 이동 기능도 제공합니다.

하지만 추천 결과가 사용자가 모르는 곡 위주로 구성될 수 있어, 사용자의 실제 취향이나 이미 보유한 플레이리스트를 충분히 반영하지 못하는 아쉬움이 있습니다.

## 핵심 아이디어

사용자가 입력한 무드, 상황, 장르에 맞춰 플레이리스트를 생성하되, 다음 두 가지 방식 중 선택할 수 있도록 합니다.

1. 사용자의 기존 플레이리스트 안에서 곡을 선정
2. 사용자가 모르는 새로운 곡을 추천

이를 통해 단순히 임의의 곡을 추천하는 것이 아니라, 사용자의 음악 취향과 현재 상황을 함께 반영한 플레이리스트를 만들 수 있습니다.

## 주요 기능

- 무드, 상황, 장르 기반 플레이리스트 생성
- 사용자의 기존 플레이리스트에서 조건에 맞는 곡 선정
- 새로운 곡 추천 여부 선택
- 특정 곡과 비슷한 유사곡 찾기
- 가사 일부를 입력하면 해당 가사가 포함된 곡 찾기
- 곡을 들려주면 제목과 가수를 찾아주는 음악 인식 기능

## GitHub 기반 구현 범위

### 초기 구현 가능 기능

- 정적 웹페이지 UI 제작
- 사용자가 직접 곡 목록 입력
- CSV 또는 JSON 파일로 플레이리스트 불러오기
- 브라우저 로컬 저장소에 플레이리스트 저장
- 무드, 상황, 장르 태그를 기준으로 곡 필터링
- 선택된 조건에 맞는 플레이리스트 생성
- GitHub Pages를 통한 배포

### 외부 API가 필요한 기능

- Spotify, FLO 등 음악 플랫폼 계정 연동
- 실제 스트리밍 서비스의 플레이리스트 불러오기
- 가사 기반 곡 검색
- 특정 곡의 유사곡 추천
- 음악을 들려주면 곡을 찾아주는 기능

이 기능들은 초기 버전에서는 직접 구현하지 않고, 추후 외부 API를 사용할 수 있을 때 확장 기능으로 추가합니다.

## 차별점

- 사용자의 기존 플레이리스트를 활용해 더 익숙하고 취향에 맞는 곡을 추천
- 새로운 곡 추천과 기존 곡 재구성을 사용자가 직접 선택 가능
- 무드, 상황, 장르를 함께 고려해 더 구체적인 플레이리스트 생성 가능
- 서버 없이 GitHub Pages만으로 실행 가능한 가벼운 웹앱 구조
