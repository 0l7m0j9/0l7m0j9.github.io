# Project: Playlist Maker

## 프로젝트 개요

사용자가 이미 듣고 있는 플레이리스트를 바탕으로, 무드, 상황, 장르 조건에 맞는 곡을 골라 새로운 플레이리스트를 만들어 주는 서비스입니다.

기존 플레이리스트 생성 서비스는 새로운 곡 추천에 초점이 맞춰져 있는 경우가 많습니다. 이 프로젝트는 사용자의 기존 취향을 먼저 활용하고, 필요할 경우 새로운 곡 추천으로 확장하는 방향을 목표로 합니다.

## 문제 인식

### CSV 또는 JSON 파일 업로드의 한계

초기 구현안에는 CSV 또는 JSON 파일로 플레이리스트를 불러오는 방식이 포함되어 있었습니다. 하지만 일반 사용자가 자신의 플레이리스트를 CSV/JSON으로 직접 준비하는 과정은 진입 장벽이 높습니다.

따라서 사용자 친화적인 시나리오를 우선 고려합니다. 예를 들어 사용자가 YouTube 플레이리스트 링크를 입력하면, 서비스가 해당 플레이리스트를 분석해 조건에 맞는 곡을 골라 주는 흐름이 더 자연스럽습니다.

### 곡 태깅 방식의 한계

무드, 상황, 장르 태그를 기준으로 곡을 필터링하려면 각 곡에 대한 태그 데이터가 필요합니다. 사람이 직접 곡마다 태그를 붙이는 방식은 확장성이 낮고, 플레이리스트 규모가 커질수록 유지하기 어렵습니다.

장기적으로는 AI를 활용해 곡의 메타데이터를 자동 생성하는 방향이 적합합니다. 다만 AI에게 어떤 입력을 제공할지에 대한 구체적인 설계가 필요합니다.

가능한 입력 예시는 다음과 같습니다.

- 곡 제목, 아티스트, 앨범 정보
- 노래 가사
- BPM, 비트, 에너지, 분위기 등 오디오 특징
- MP3 또는 음원 일부
- YouTube 설명, 댓글, 카테고리 등 외부 맥락 정보

이 중 GitHub Pages 기반 초기 버전에서는 API key 노출 문제가 있으므로 실제 AI API 연동은 바로 구현하지 않습니다.

## 개발 방향

초기 버전은 GitHub Pages에서 실행 가능한 정적 웹 앱으로 구현합니다. 별도의 백엔드 서버를 두지 않고, 사용자의 입력과 결과 화면을 중심으로 MVP를 구성합니다.

AI 태깅, YouTube API 연동, Spotify/FLO 연동 등 API key가 필요한 기능은 실제 연동 대신 fake door MVP로 검증합니다. 즉, 사용자가 실제 서비스 흐름처럼 행동할 수 있는 화면과 시나리오를 제공하되, 내부 데이터는 미리 준비된 샘플 데이터로 처리합니다.

## MVP 시나리오

### 시나리오 1: YouTube 플레이리스트 링크 입력

사용자가 YouTube 플레이리스트 링크를 입력합니다.

서비스는 실제 YouTube API를 호출하지 않고, 미리 준비된 샘플 플레이리스트를 불러온 것처럼 보여 줍니다. 이후 사용자는 무드, 상황, 장르 조건을 선택하고, 그 조건에 맞는 결과 플레이리스트를 확인합니다.

검증 목표:

- 사용자가 CSV/JSON보다 YouTube 링크 입력 방식을 더 자연스럽게 느끼는지 확인
- 플레이리스트 기반 필터링 흐름이 이해하기 쉬운지 확인
- 사용자가 어떤 조건으로 곡을 고르고 싶어 하는지 확인

### 시나리오 2: 상황 기반 플레이리스트 생성

사용자가 "운동", "공부", "퇴근길", "파티" 같은 상황을 선택합니다.

서비스는 샘플 곡 데이터에 미리 부여된 상황 태그를 기준으로 곡을 필터링하고, 결과 플레이리스트를 생성합니다.

검증 목표:

- 사용자가 장르보다 상황 중심 탐색을 선호하는지 확인
- 상황 태그가 실제 곡 선택에 유용한지 확인
- 결과가 충분히 납득 가능한지 확인

### 시나리오 3: AI 태깅 결과 미리보기

사용자가 곡을 선택하면, 서비스는 AI가 분석한 것처럼 무드, 상황, 장르 태그를 보여 줍니다.

초기 버전에서는 실제 AI 분석을 수행하지 않고, 샘플 데이터에 저장된 태그와 설명을 사용합니다.

검증 목표:

- 사용자가 AI 태깅 결과를 신뢰하는지 확인
- 어떤 설명이 있어야 태그 결과를 납득하는지 확인
- 향후 실제 AI 연동 시 필요한 입력 데이터 범위를 파악

## 주요 기능

### 초기 구현 기능

- YouTube 플레이리스트 링크 입력 UI
- 샘플 플레이리스트 로딩
- 무드, 상황, 장르 태그 선택
- 선택 조건에 따른 곡 필터링
- 결과 플레이리스트 생성
- 곡별 태그 및 간단한 분석 설명 표시
- GitHub Pages 배포

### 초기 버전에서 제외하는 기능

- 실제 YouTube API 연동
- 실제 Spotify, FLO 등 음악 플랫폼 연동
- 실제 AI API를 통한 곡 분석
- MP3 업로드 및 오디오 분석
- 사용자 계정 및 서버 저장 기능
- CSV/JSON 파일 업로드 중심의 사용자 흐름

## AI 태깅 확장 방향

향후 실제 AI를 도입할 경우, 다음 순서로 확장할 수 있습니다.

1. 곡 제목, 아티스트, 가사 기반 태깅
2. BPM, 에너지, 댄서빌리티 등 오디오 특징 추가
3. 짧은 음원 클립 또는 MP3 기반 분석
4. 사용자 피드백을 반영한 태그 보정

초기에는 가사와 기본 메타데이터를 활용하는 방식이 가장 현실적입니다. MP3 기반 분석은 정확도는 높일 수 있지만, 저작권, 파일 처리, 저장, 비용 문제가 커지므로 후순위로 둡니다.

## GitHub Pages 기반 구현 범위

GitHub Pages에서는 API key를 안전하게 숨기기 어렵기 때문에, 외부 API 호출이 필요한 기능은 실제 구현하지 않습니다.

대신 다음과 같은 방식으로 MVP를 구성합니다.

- 샘플 YouTube 플레이리스트 데이터 준비
- 샘플 곡별 태그 데이터 준비
- AI 분석처럼 보이는 설명 데이터 준비
- 사용자가 실제 서비스처럼 입력하고 결과를 확인할 수 있는 화면 구현

이 방식은 완전한 기능 구현보다 빠르게 사용자 반응을 확인하는 데 초점을 둡니다.

## 차별점

- 사용자가 이미 듣는 플레이리스트를 출발점으로 삼음
- CSV/JSON보다 친숙한 YouTube 플레이리스트 입력 흐름을 우선 고려
- 무드, 상황, 장르를 함께 반영한 필터링 경험 제공
- AI 태깅 기능은 실제 연동 전 fake door MVP로 검증
- GitHub Pages만으로 실행 가능한 가벼운 초기 구조

## 향후 확장 방향

- YouTube API 연동
- Spotify, FLO 등 음악 플랫폼 연동
- 실제 AI 기반 곡 태깅
- 가사 검색 및 분석 기능 고도화
- 오디오 특징 추출 기반 추천
- 사용자 피드백 기반 추천 품질 개선
