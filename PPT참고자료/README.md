# Playlist Maker

## 프로젝트 개요

Spotify 계정으로 로그인해 내 플레이리스트를 불러오고, 무드·상황·장르·가수·재생 시간·원곡 대 추천곡 비율까지 조건에 맞춰 새로운 플레이리스트를 만드는 서비스입니다. 별도 백엔드 없이 GitHub Pages에서 정적 파일(`index.html`) 하나로 동작하며, Spotify Web API와 브라우저에서 직접 통신합니다.

기존 플레이리스트 생성 서비스는 새로운 곡 추천에 초점이 맞춰져 있는 경우가 많습니다. 이 프로젝트는 사용자가 이미 듣고 있는 플레이리스트를 먼저 활용하고, 관련 아티스트의 곡을 추천곡으로 자연스럽게 섞어주는 방향을 목표로 합니다. 원곡:추천곡 비율(7:3, 5:5 등)은 사용자가 직접 정할 수 있습니다.

> 초기에는 YouTube 링크 입력, 여러 음악 앱 연동, 마이크 음악인식, 가사 검색 등을 흉내만 내는 Fake Door MVP(샘플 데이터 기반)로 시작했지만, 현재는 **Spotify 실제 로그인과 실제 API 데이터만** 사용하는 버전으로 전면 개편되었습니다.

## 주요 기능

- **Spotify 로그인 (OAuth 2.0 PKCE)**: 별도 서버나 Client Secret 없이 브라우저에서 안전하게 로그인. 로그인 버튼을 누르면 Spotify가 직접 호스팅하는 로그인 화면으로 이동합니다.
- **내 플레이리스트 불러오기 + 믹스 비율 커스터마이즈**: 로그인 후 내 플레이리스트를 선택하면 원곡에 관련 아티스트 추천곡을 자연스럽게(랜덤 가중치 방식) 섞어 새 목록을 구성합니다. 원곡:추천곡 비율은 내 곡만(10:0)부터 5:5까지 직접 선택할 수 있고, 이미 불러온 뒤 비율을 바꿔도 API를 다시 부르지 않고 즉시 재구성합니다.
- **메인 화면 통합 검색/필터**: 별도 단계 없이 첫 화면에서 바로 검색·필터가 가능합니다.
  - 플레이리스트를 불러온 상태: 그 안에서 곡 제목/아티스트/무드로 검색, 무드·상황·장르·가수 칩으로 필터링(추론 알고리즘 포함)
  - 아직 아무것도 안 불러온 상태: 검색어나 무드/상황/장르 선택 시 **Spotify 카탈로그를 직접 검색**해서 결과를 보여줌
- **곡 선택 → 비슷한 곡, 아티스트 선택 → 그 아티스트 곡**: 카탈로그 검색 결과에서 곡을 하나 선택하면 같은 아티스트의 다른 곡 + 같은 장르의 곡을 모아 "비슷한 곡"으로 전환합니다. 검색어가 아티스트명과 겹치면 아티스트 후보도 함께 표시되고, 클릭하면 그 아티스트의 곡들로 결과가 전환됩니다. (Spotify 추천 API가 막혀 있어 아티스트·장르 조합으로 근사합니다.)
- **상황 → 무드 → 장르 추론 알고리즘**: 상황(카페, 운동, 드라이브 등)을 고르면 연관 무드를, 무드를 고르면 연관 장르를 자동으로 추론해 점수 기반으로 곡을 정렬합니다.
- **재생 시간 선택**: 원하는 재생 시간(15분~2시간)을 고르면, 선택한 믹스 비율을 유지하면서 목표 시간까지 원곡과 추천곡을 채웁니다.
- **결과 셔플**: "플레이리스트 만들기"를 누를 때마다 최종 곡 순서를 무작위로 섞어 매번 다른 순서로 완성됩니다.
- **앨범 커버 표시**: 곡 목록, 결과 화면, 상세 모달, 연관곡 전부 Spotify 앨범 커버 이미지를 표시합니다.
- **플레이리스트 이름 직접 입력**: 결과 화면에서 저장할 플레이리스트 이름을 직접 입력하거나(선택 조건 기반 자동 제안) 그대로 저장할 수 있습니다.
- **Spotify에 실제 저장**: 완성된 플레이리스트를 "Spotify에 저장" 버튼으로 내 계정에 실제 플레이리스트로 생성합니다.
- **더 보기(페이지네이션)**: 곡이 많을 때 한 번에 다 그리지 않고 20곡씩 "더 보기"로 펼쳐봅니다.

## 실행 방법 (배포자용 설정)

이 프로젝트는 빌드 과정이 필요 없는 순수 정적 HTML/CSS/JS입니다. 다만 Spotify 로그인이 동작하려면 배포 전 아래 설정이 필요합니다.

1. [Spotify Developer Dashboard](https://developer.spotify.com/dashboard)에서 앱 생성
   - API는 **Web API**만 선택
   - Redirect URIs에 실제 배포 주소 등록 (예: `https://0l7m0j9.github.io/`, `https://0l7m0j9.github.io/index.html`, 로컬 테스트용 `http://127.0.0.1:5500/index.html`)
2. 발급받은 **Client ID**를 `index.html`의 `SPOTIFY_CLIENT_ID` 상수에 입력 (Client Secret은 필요 없음 — PKCE 방식)
3. GitHub Pages로 배포: 이 저장소가 `사용자명.github.io` 형식이므로 `main` 브랜치에 푸시하면 `https://사용자명.github.io/`에서 자동으로 서비스됩니다.

### ⚠️ Spotify Premium 계정 필요

2026년 2월 Spotify 정책 변경으로, Development Mode 앱을 만들려면 **개발자(앱 소유자) 계정이 Spotify Premium**이어야 합니다. 또한 Development Mode는 다음 제약이 있습니다.

- 계정당 앱(Client ID) 1개만 생성 가능
- 앱당 로그인 가능한 사용자 최대 5명 (Dashboard의 User Management에서 이메일로 등록)
- 개발자의 Premium 구독이 끊기면 앱이 즉시 동작 중단

## 기술 스택 / 구조

- **단일 정적 파일** (`index.html`): HTML/CSS/JS가 한 파일에 포함, 별도 빌드 도구 없음
- **인증**: Spotify OAuth 2.0 Authorization Code + PKCE (브라우저에서 안전하게 로그인, 토큰은 `localStorage`에 보관)
- **데이터 소스**: Spotify Web API (`/me/playlists`, `/playlists/{id}/items`, `/search`, `/artists/{id}`, `/me/playlists` POST 등) — 모두 실제 API, 샘플/더미 데이터 없음
- **무드/장르 태깅**: Spotify가 오디오 특성(BPM, 에너지 등) API를 더 이상 공개하지 않아, 아티스트 실제 장르 정보 + 제목/아티스트 텍스트 휴리스틱 + 곡별 다양성 보정을 조합해 추정
- **호스팅**: GitHub Pages (백엔드 서버 없음)

## 알려진 제약

- Spotify Web API의 오디오 특성(`audio-features`), 추천(`recommendations`), 배치 조회 등 다수 엔드포인트가 2024~2026년 사이 단계적으로 제한/제거되어, 무드 태깅은 완벽한 AI 분석이 아닌 근사치입니다.
- 가사 검색·마이크 음악인식 기능은 Spotify Web API가 가사 데이터를 제공하지 않아 지원하지 않습니다.
- Development Mode 제약(최대 5명, Premium 필요)으로 인해 불특정 다수를 대상으로 한 공개 서비스에는 적합하지 않습니다. 더 많은 사용자를 지원하려면 Spotify의 Extended Quota Mode 심사를 받아야 합니다.

## 향후 확장 방향

- Extended Quota Mode 신청을 통한 다중 사용자 지원
- 재생 시간 채우기 로직 고도화 (BPM/에너지 흐름 고려)
- 결과 플레이리스트 커버 이미지 커스터마이징
- 필터 조합 히스토리 저장 및 재사용
