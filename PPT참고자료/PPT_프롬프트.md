슬라이드 순서와 각 슬라이드 내용은 아래와 같아.

# 1. 기존 서비스와 Playlist Maker 비교
- 기존 서비스: 알고리즘이 고른 낯선 곡 위주 추천, 새 곡 비중이 대부분/전부, 장르 정도로만 조건 설정
- Playlist Maker: 내 Spotify 플레이리스트를 출발점으로 삼음, 원곡:추천곡 비율을 사용자가 직접 설정(내 곡만~5:5), 무드·상황·장르·가수·재생시간까지 조합 설정, 만들 때마다 곡 순서 무작위 셔플, 곡/아티스트를 선택하면 비슷한 곡·그 아티스트 곡으로 즉시 전환
- 핵심 문장: "이미 듣는 곡에서, 딱 맞는 새 플레이리스트를"

# 2. Playlist Maker 제작 과정
- 1단계 기획: Fake Door MVP로 실제 API 연동 없이 화면 흐름만 먼저 검증 (YouTube 링크 입력 UI, 여러 음악 앱 연동인 척하는 버튼, 마이크 음악인식 가짜 시뮬레이션, 가사로 곡 찾기 가짜 DB, 샘플곡 30~50곡에 수동 태깅)
- 2단계 전환: 샘플 데이터로는 검증에 한계가 있어 실제 Spotify 연동으로 전면 개편 결정
- 3단계 구축: OAuth 2.0 PKCE 로그인 구현(서버 없이), 내 플레이리스트 불러오기+추천곡 믹스 구현, 가짜 기능(YouTube 링크, 여러 플랫폼 fake 연동, 마이크 인식, 가사 검색) 전부 제거
- 4단계 UX 개선: 검색 기능을 하위 메뉴에서 메인 화면으로 이동, 재생시간 선택 위치를 하단에서 상단으로 이동, 원곡:추천곡 고정비율(7:3)을 사용자가 직접 선택하는 방식으로 변경, 검색 결과 중복/부정확 문제 발견 후 개선, 결과 순서 고정에서 매번 무작위 셔플로 변경
- 5단계 브랜딩: 로고/파비콘 적용, README 및 발표자료 정리

# 3. Playlist Maker 제작 과정 및 난관
- 2024~2026년 누적: Spotify가 오디오 특성(BPM·에너지) API, 추천(Recommendations) API, 아티스트 배치 조회 API를 단계적으로 폐지 → 오디오 분석 기반 정밀 태깅 대신 아티스트 장르 정보+텍스트 키워드 휴리스틱으로 자체 추정 로직 구현
- 2026년 2월 11일: Spotify Development Mode 정책 대개편 발표 — 앱 소유자가 반드시 Premium 계정이어야 함, 앱 1개 제한, 로그인 최대 5명 제한. 실제로 Premium을 구독하고서야 앱 생성 가능했음
- 같은 정책으로 엔드포인트 경로 변경: /playlists/{id}/tracks가 /items로, /users/{id}/playlists가 /me/playlists로 변경, 아티스트 배치 조회(GET /artists) 폐지 → 기존 코드가 갑자기 403/빈 응답 반환, 원인 추적 후 마이그레이션
- 개발 중 발견한 문제 1: 무드/상황 필터와 검색어를 같이 Spotify에 보내면 제목은 안 맞고 장르만 맞는 엉뚱한 곡이 섞여 나옴 → 검색어 있을 때는 텍스트만으로 검색하도록 분리
- 개발 중 발견한 문제 2: 같은 곡의 싱글판·리마스터판이 다른 트랙 ID로 중복 나열됨 → 제목+아티스트 기준으로 추가 중복 제거
- 개발 중 발견한 문제 3: 재생시간을 채우려 해도 추천곡이 부족함 → 추천곡을 필요량의 몇 배(최대 80곡)로 미리 넉넉히 확보해두는 방식으로 개선

# 4. Playlist Maker 기술 구조
- 구조: 단일 정적 파일(index.html) 하나에 HTML/CSS/JS 전부 포함, 빌드 도구 없음
- 인증: OAuth 2.0 Authorization Code + PKCE 방식, Client Secret 불필요, 브라우저에서 Spotify와 직접 통신(중간 서버 없음)
- 토큰 저장: localStorage에 Access/Refresh Token 보관
- 데이터: 전부 실제 Spotify Web API, 샘플/더미 데이터 없음
- 호스팅: GitHub Pages, 사용자명.github.io 저장소에 main 브랜치 푸시만으로 자동 배포
- 실제 호출 엔드포인트: GET /me/playlists, GET /playlists/{id}/items, GET /search, GET /artists/{id}, POST /me/playlists, POST /playlists/{id}/items
- 강조 포인트: 서버 비용 0원, 배포는 git push 한 줄, 대신 Client Secret을 못 숨기기 때문에 PKCE 방식을 쓸 수밖에 없었다는 인과관계

# 5. 사용자의 Playlist Maker 사용 방법
- 1단계: "Spotify로 로그인" 버튼 클릭 → Spotify 자체 로그인 화면에서 로그인 → 자동으로 사이트 복귀
- 2단계: 내 플레이리스트를 선택해 비율(예: 7:3)을 정해 불러오거나, 곡/아티스트를 검색해서 Spotify 카탈로그를 바로 탐색
- 3단계: 무드·상황·장르·가수 칩을 클릭해 조건을 좁히고, 재생시간(15분~2시간)을 선택하면 그 시간에 맞게 곡 수가 자동 조정됨
- 4단계: 검색 결과에서 곡을 선택하면 비슷한 곡으로, 아티스트를 선택하면 그 아티스트의 곡으로 화면이 전환되어 계속 탐색 가능
- 5단계: "플레이리스트 만들기" → 결과 확인 → 이름을 직접 입력하거나 자동 제안된 이름 사용 → "Spotify에 저장" → 내 Spotify 앱에서 바로 확인

# 6. Playlist Maker 기능 ① — Spotify 로그인 & 플레이리스트 불러오기
- PKCE 로그인: Client Secret 없이 브라우저에서 안전하게 인증
- 로그인 유지: 토큰을 localStorage에 저장해 새로고침해도 로그인 상태 유지
- 믹스 비율 커스터마이즈: 내 곡만(10:0)/8:2/7:3(기본)/6:4/5:5 중 선택 가능, 비율을 바꿔도 API 재호출 없이 즉시 재구성
- 추천곡은 내 플레이리스트 상위 아티스트 12명 × 최대 10곡, 최대 80곡까지 미리 확보
- 원곡 순서는 유지하면서 추천곡을 비율대로 자연스럽게(가중치 랜덤) 끼워 넣는 믹스 방식

# 7. Playlist Maker 기능 ② — 통합 검색 (비슷한 곡·아티스트 선택)
- 플레이리스트를 불러온 상태면 그 안에서, 안 불러온 상태면 Spotify 카탈로그를 직접 검색
- 곡을 선택하면 같은 아티스트의 다른 곡 + 같은 장르의 곡을 모아 "비슷한 곡"으로 전환
- 검색어가 아티스트명과 겹치면 아티스트 후보가 함께 뜨고, 클릭하면 그 아티스트의 곡으로 전환
- 검색어가 있을 때는 텍스트로만 검색해 정확도를 높이고, 중복곡은 트랙ID+제목/아티스트 기준으로 이중 제거
- 한계: Spotify 공식 추천 API가 막혀 있어 아티스트·장르 조합으로 근사한 결과이며 완벽한 AI 추천은 아님

# 8. Playlist Maker 기능 ③ — 무드·상황·장르 추론 알고리즘
- 상황을 고르면 연관 무드를, 무드를 고르면 연관 장르를 자동으로 추론 (예: 상황 "카페" → 무드 여유로운/따뜻한/설레는 → 장르 힙합/R&B/인디)
- 가수 직접선택 +10점, 무드 직접선택 +4점, 무드 추론 +2점, 장르 직접선택 +3점, 장르 추론 +1점을 합산해 점수 높은 순으로 정렬
- 직접 선택한 조건은 실선 칩, 알고리즘이 자동으로 추론한 연관 항목은 점선 칩으로 구분 표시해 추론 과정을 투명하게 공개
- 오디오 특성 API가 막혀 아티스트 장르 정보+텍스트 키워드+다양성 보정을 조합한 근사치 태깅

# 9. Playlist Maker 기능 ④ — 재생 시간 맞춤 & 결과 셔플
- 재생시간 선택지: 15분/30분/45분/1시간/1시간30분/2시간/제한없음
- 선택한 믹스 비율을 유지한 채 목표 시간까지 원곡·추천곡을 채우며, 선택 즉시 화면에 실시간 반영
- 플레이리스트를 만들 때마다 최종 곡 순서를 무작위로 재배치(필터링 로직은 그대로, 순서만 매번 새롭게)
- Spotify에 저장할 때도 화면에 보여준 셔플된 순서 그대로 저장

# 10. Playlist Maker 기능 ⑤ — 결과 화면 (앨범 커버·이름 입력·저장)
- 곡 목록, 결과 카드, 상세 모달, 연관곡까지 전부 실제 Spotify 앨범 커버 이미지 표시
- 결과 화면에서 플레이리스트 이름을 직접 입력하거나 비워두면 선택 조건 기반으로 자동 제안
- "Spotify에 저장" 버튼으로 내 계정에 실제 플레이리스트 생성, 저장 진행 상황 표시, 완료 후 새 탭으로 자동 오픈
- 곡이 많을 때 20곡씩 "더 보기"로 펼치는 페이지네이션

# 11. Playlist Maker 한계
- 태깅은 근사치: 오디오 특성 API 폐지로 무드/장르가 완벽한 AI 분석이 아닌 추정치
- "비슷한 곡"도 근사치: 공식 추천 API가 막혀 아티스트·장르 조합으로 대체한 것이지 진짜 추천 엔진이 아님
- 가사 검색·마이크 인식 미지원: Spotify Web API가 가사 데이터를 제공하지 않음
- 최대 5명 제한: Development Mode 제약으로 앱 1개, 로그인 최대 5명, 개발자 Premium 필수
- 공개 서비스로는 아직 부적합: 불특정 다수 대상으로 넓히려면 Spotify의 별도 심사(Extended Quota Mode) 필요

# 12. Playlist Maker 개선 방안
- 태깅 근사치 문제 → 사용자가 곡에 직접 태그를 수정·피드백할 수 있는 기능 검토
- "비슷한 곡" 정확도 → 아티스트·장르 조합 로직 고도화 및 외부 음악 메타데이터 API 보강 검토
- 가사 검색 미지원 → Musixmatch 등 외부 가사 API 연동 검토
- 최대 5명 제한 → Spotify Extended Quota Mode 심사 신청으로 사용자 수 제한 해제
- 재생시간 채우기 단순함 → BPM/에너지 흐름을 고려한 곡 배치 로직 고도화
