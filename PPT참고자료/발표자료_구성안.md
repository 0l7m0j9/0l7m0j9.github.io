# Playlist Maker — 발표 슬라이드 구성안 (PPT용 텍스트)

각 슬라이드를 PowerPoint에 그대로 옮기면 됩니다. "말할 내용(스피커 노트)"은 발표 시 참고용입니다.

## 슬라이드 1. 타이틀

**제목**: 이미 듣는 곡에서, 딱 맞는 새 플레이리스트를
**부제**: Playlist Maker — Spotify 기반 스마트 플레이리스트 생성 서비스
**하단**: 4조 · Team Project

배지: 실제 Spotify Web API 사용 / 백엔드 서버 없음 / GitHub Pages 배포

---

## 슬라이드 2. 문제 인식

**제목**: 대부분의 추천 서비스는 "새 곡 추천"에만 초점이 맞춰져 있다

| 기존 서비스 | Playlist Maker |
|---|---|
| 알고리즘이 고른 낯선 곡 위주 추천 | 내 플레이리스트를 출발점으로 삼음 |
| 내가 이미 좋아하는 곡은 활용 안 함 | 원곡:추천곡 비율을 직접 조절 (7:3~5:5 등) |
| 무드 조건 선택 폭이 좁음 | 무드·상황·장르·재생 시간까지 조건화 |

**말할 내용**: 왜 이 프로젝트를 시작했는지 — 기존 플레이리스트 추천 서비스의 한계와 우리가 다르게 접근한 지점.

---

## 슬라이드 3. 핵심 흐름

**제목**: 세 단계로 끝나는 플레이리스트 생성

`Spotify 로그인` → `비율·시간·무드/상황 선택` → `완성 & Spotify 저장`

**말할 내용**: 별도 앱 심사나 계정가입 없이 로그인 한 번이면 시작. 방문자는 Client ID를 볼 일도 없음(코드에 미리 심어둔 값으로 처리).

---

## 슬라이드 4. 기능 1 — Spotify 로그인

**제목**: 서버 없이 안전한 Spotify 로그인

- OAuth 2.0 Authorization Code + PKCE 방식
- **PKCE**: code_verifier/code_challenge로 Client Secret 없이 안전하게 토큰 교환
- **로컬 저장**: Access/Refresh Token을 localStorage에 보관, 새로고침해도 로그인 유지
- **즉시 로그아웃**: 토큰·캐시 전부 삭제

---

## 슬라이드 5. 기능 2 — 믹스 비율 커스터마이즈

**제목**: 내가 정하는 믹스 비율

- 옵션: 내 곡만(10:0) / 8:2 / **7:3(기본값)** / 6:4 / 5:5
- 이미 곡을 불러온 뒤 비율을 바꿔도 API 재호출 없이 즉시 재구성
- 추천곡 풀을 미리 넉넉히(상위 아티스트 12명 × 최대 10곡, 최대 80곡) 확보해둠

---

## 슬라이드 6. 기능 3 — 무드→상황→장르 추론

**제목**: 상황 → 무드 → 장르, 계층형 추론

예시: `☕ 상황: 카페` → `여유로운·따뜻한·설레는` → `인디·팝·R&B`

**말할 내용**: 상황을 고르면 연관 무드를, 무드는 다시 연관 장르를 자동 추론해 점수 기반으로 정렬. 플레이리스트를 안 불러온 상태에서도 이 추론으로 Spotify 카탈로그를 즉시 검색.

---

## 슬라이드 7. 기능 4 — 검색 UX

**제목**: 검색은 탐색의 시작점

- **곡 제목 검색**: 검색 → 목록에서 곡 선택 → 같은 아티스트 곡 + 같은 장르 곡을 모아 "비슷한 곡"으로 전환
- **아티스트 검색**: 검색어가 아티스트명과 겹치면 아티스트 칩 표시 → 클릭 시 그 아티스트 곡으로 전환

**말할 내용**: Spotify 공식 추천 API가 막혀 있어, 아티스트·장르 조합으로 "비슷한 곡"을 자체 근사.

---

## 슬라이드 8. 기능 5 — 재생 시간 + 셔플

**제목**: 재생 시간에 맞춰 자동으로 채우기

- 15분 / 30분 / 45분 / 1시간 / 1시간 30분 / 2시간 중 선택
- 선택한 믹스 비율을 유지한 채 목표 시간까지 채움
- "플레이리스트 만들기" 누를 때마다 최종 순서를 무작위로 셔플 (필터링 결과는 그대로, 순서만 매번 다르게)

---

## 슬라이드 9. 기술 구조

**제목**: 서버 없이, 파일 하나로

1. **정적 파일 1개** — index.html 하나에 HTML/CSS/JS 전부 포함, 빌드 도구 없음
2. **PKCE 인증** — 토큰은 localStorage 보관, Secret 없이 브라우저에서 완결
3. **GitHub Pages** — 사용자명.github.io 저장소, main 브랜치 푸시만으로 자동 배포

사용 엔드포인트: `/me/playlists`, `/playlists/{id}/items`, `/search`, `/artists/{id}`, `POST /me/playlists`

## 슬라이드 10. 개발 중 마주친 난관 (스토리텔링 슬라이드)

**제목**: Spotify가 문을 계속 좁혀왔다

- **2024~2026**: 오디오 특성(BPM/에너지)·추천 API 단계적 폐지 → 장르·텍스트 휴리스틱으로 자체 태깅 로직 구현
- **2026.02.11**: Development Mode 정책 변경 — 앱 소유자 Premium 계정 필수, 앱 1개, 로그인 최대 5명 제한. 엔드포인트 경로도 변경(`/playlists/{id}/tracks`→`/items`, `/users/{id}/playlists`→`/me/playlists`)
- **대응**: 바뀐 엔드포인트로 마이그레이션, 배치 조회 제거분은 개별 호출로 대체, 검색 결과 중복 제거 로직 추가

**말할 내용**: 실제 서비스 정책이 개발 중 바뀌는 걸 어떻게 대응했는지 — 좋은 문제해결 사례로 어필할 수 있는 슬라이드.
